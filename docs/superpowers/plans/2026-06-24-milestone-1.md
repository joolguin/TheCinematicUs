# Milestone 1 Implementation Plan — bugs + privacidad + quick wins

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cerrar los bugs latentes del pozo/cache, volver la privacidad de likes testeable y fail-closed, y sacar las URLs de Letterboxd y el estado de enriquecimiento de configuraciones rígidas.

**Architecture:** Backend Express/TS (ESM) con un único cliente Supabase service-role (`db.ts`). Los cambios de schema viven en `backend/db/schema.sql` como migraciones idempotentes que se ejecutan a mano en el SQL editor de Supabase. La lógica se testea con vitest mockeando `./db.js` y `./tmdb.js`. La privacidad se valida con un test de integración que usa la anon key real.

**Tech Stack:** Node 24 (alpine), TypeScript ESM (imports con extensión `.js`), Express, `@supabase/supabase-js`, vitest. TMDB como fuente de enriquecimiento.

## Global Constraints

- **Todo corre en Docker. Nunca `npm`/`node` en el host.** Comando de test del backend:
  `docker compose run --rm backend npm --workspace backend test`
  Para un archivo: `docker compose run --rm backend npm --workspace backend test -- src/<archivo>.test.ts`
- **ESM:** todos los imports internos llevan extensión `.js` (ej. `import { supabase } from './db.js'`).
- **Migraciones idempotentes:** todo DDL nuevo va al final de `backend/db/schema.sql`, con `if not exists` / `drop ... if exists`, y además se refleja en el `create table` de cabecera para DB nuevas. El archivo se aplica a mano en Supabase; no hay runner de migraciones.
- **Commits sin trailer `Co-Authored-By: Claude`.**
- Mensajes de commit en español, estilo del repo (`fix(backend): ...`, `feat(backend): ...`).

### Task 1: `search_key` deja de ser `unique` (lectura tolerante a duplicados)

Insertar, justo después de calcular `uniqueIds` y **antes** del `delete`:

```ts
  // Guardia anti-degradación: comparar contra el set actual.
  const { data: current } = await supabase
    .from('watchlist_items').select('movie_id').eq('user_id', userId);
  const prevIds = new Set((current ?? []).map((r: { movie_id: string }) => r.movie_id));
  if (prevIds.size > 0) {
    const newIds = new Set(uniqueIds);
    const removed = [...prevIds].filter((id) => !newIds.has(id)).length;
    const ratio = removed / prevIds.size;
    if (ratio > MAX_REMOVAL_RATIO) {
      return {
        count: prevIds.size,
        ok: false,
        kept: true,
        error: `diff sospechoso: ${Math.round(ratio * 100)}% del pozo desaparecería; se mantiene el set anterior`,
      };
    }
  }
```

```ts
// Ventana fija de reintento de enriquecimiento. A esta escala (cientos de pelis,
// 2 usuarias) un reintento diario alcanza; no hace falta backoff exponencial.
const ENRICH_RETRY_MS = 24 * 60 * 60 * 1000;

// Reintenta enriquecer las pelis con enriched=false cuyo último intento fue hace
// más de ENRICH_RETRY_MS (o nunca). Pensado para correr desde un cron (M1.5/§7).
export async function reEnrichStale(now: Date = new Date()): Promise<{ attempted: number; enriched: number }> {
  const cutoff = new Date(now.getTime() - ENRICH_RETRY_MS).toISOString();
  const { data: stale } = await supabase
    .from('movies')
    .select('id, title, year')
    .eq('enriched', false)
    .or(`last_enrich_attempt_at.is.null,last_enrich_attempt_at.lt.${cutoff}`);

  const rows = stale ?? [];
  let enrichedCount = 0;
  for (const row of rows) {
    const m = await searchAndEnrich(row.title, row.year);
    const ts = now.toISOString();
    await supabase.from('movies').update({
      tmdb_id: m.tmdbId, title: m.title, original_title: m.originalTitle, year: m.year,
      poster_url: m.posterUrl, director: m.director, cast: m.cast, runtime: m.runtime,
      genres: m.genres, overview: m.overview, tmdb_rating: m.tmdbRating, country: m.country,
      enriched: m.enriched,
      fetched_at: m.enriched ? ts : null,
      last_enrich_attempt_at: ts,
    }).eq('id', row.id);
    if (m.enriched) enrichedCount++;
  }
  return { attempted: rows.length, enriched: enrichedCount };
}
```
